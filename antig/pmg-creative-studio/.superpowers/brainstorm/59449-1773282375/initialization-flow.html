Mask Editor Initialization — After First Extraction

How the API result becomes an editable mask

Step 1: What the API returns

extractForeground() → Replicate result
Foreground PNG
Checkerboard = transparent (alpha=0)
Subject pixels = opaque (alpha=255)
Edges = partial alpha (anti-aliased)
Alpha channel (what we care about)
White = kept by API (alpha=255)
Black = removed by API (alpha=0)
Gray edges = partial alpha

Step 2: Read the alpha channel (requires CORS proxy)

Offscreen canvas reads pixel data from the extracted PNG
extractedImageUrl
Replicate URL
→
proxyUrl()
CORS-safe
→
Offscreen canvas
drawImage() → getImageData()
→
Alpha channel array
pixel[i*4 + 3] for each pixel
This is why the proxy is required. Without CORS-safe loading, getImageData() throws SecurityError and we can't read the alpha values.

Step 3: Initialize both canvases from the alpha data

Alpha channel → mask canvas + display red tint
Hidden Mask Canvas
for each pixel:
  alpha=0 → paint black (removed)
  alpha=255 → paint white (keep)
  partial → proportional gray
Visible Display Canvas
Layer 1: original image (full)
Layer 2: red tint only where alpha=0
Subject area = clear (no red)
Background area = red tinted

Step 4: How the red tint overlay is built

Pixel-level red tint generation
// Create a raster overlay image from the alpha channel
// This becomes a FabricImage on the display canvas

tintCanvas = new OffscreenCanvas(naturalWidth, naturalHeight)
tintData = tintCanvas.getImageData()

for each pixel i:
  extractedAlpha = alphaChannel[i]

  // Where API removed background (alpha=0) → show red tint
  // Where API kept subject (alpha=255) → fully transparent (no tint)

  tintData[i*4 + 0] = 255    // R = red
  tintData[i*4 + 1] = 0     // G = 0
  tintData[i*4 + 2] = 0     // B = 0
  tintData[i*4 + 3] = Math.round((1 - extractedAlpha/255) * 102)
  // alpha=0 (removed) → tint alpha=102 (~40% red)
  // alpha=255 (kept) → tint alpha=0 (no red)
  // partial alpha → proportional red tint

// Convert to image, load as FabricImage on display canvas
tintCanvas.putImageData(tintData)
tintBlob = tintCanvas.toBlob('image/png')
tintUrl = URL.createObjectURL(tintBlob)
fabricTint = new FabricImage(tintUrl) → add to display canvas

Summary: The Full Init Sequence

1
Load extractedImageUrl through proxy → draw on offscreen canvas → getImageData()
2
Read alpha channel pixel-by-pixel → this IS the API's edge detection result
3
Initialize mask canvas: alpha→luminance (0→black, 255→white, partial→gray)
4
Generate red tint raster: inverted alpha → red overlay (0→red, 255→clear)
5
Display canvas: original image (bg) + red tint FabricImage (overlay) → ready for painting
Key insight: The API's edge detection is encoded in the alpha channel of the PNG it returns. We read that alpha channel to build both the visual red tint and the data mask. The edges are preserved exactly — anti-aliased edges in the API result become proportional gray values in the mask and proportional red tint on the display.
